Quick Start

  • i → Insert 模式，按 ESC 回到 Normal 模式.
  • x → 删当前光标所在的一个字符。
  • :wq → 存盘 + 退出 (:w 存盘, :q 退出) （陈皓注：:w 后可以跟文件名）
  • dd → 删除当前行，并把删除的行存到剪贴板里
  • p → 粘贴剪贴板

基本使用

在 Vim 中有 4 种编辑模式，分别是：普通模式、插入模式、可视模式和命令行模式。使用 Vim 进行编辑就需要熟练的在各个模式之间进行切换。

普通模式

  • 进入编辑器的时候默认为普通模式，用 h/j/k/l 来对光标进行移动

    • 左右两边的 h/l 代表左右移动，而食指在键盘的默认位置 j即为向下，反之 k 为向上
    • 普通模式下输入是一个命令，可以指定次数，例如向下移动 10 行——连续按 10 次 j或直接输入 10j

  • 更高效的光标移动命令：

    • ^/$：跳到行首/行尾（正则表达式的内容）
    • w/b：跳到下一个单词头/跳到上一个单词头
    • 快捷键： ctrl + a 或者 ctrl + e 来跳到行首或者行尾（windows某些编辑器可能不生效）

  • f{char}/F{char}：跳到下一个字符为 char 的位置/跳到上一个字符为 char 的位置——实现向前/向后搜索某个字符，然后跳过去

以上的各种跳转方式可以满足绝大多数跳转的场景，熟练使用之后基本上就不要再通过鼠标去移动光标了

“画家在休息时不会把画笔放在画布上。对 Vim 而言也是这样，普通模式就是 Vim 的自然放松状态，其名字已经寓示了这一点。” ——《Vim实用技巧》

“谁说一定要切换到插入模式才行？我们可以重新调整已有代码的格式，复制它们，移动其位置，或是删除它们。在普通模式中，我们有众多的工具可以利用。”——《Vim实用技巧》

插入模式

  • 在插入模式进行插入字符、换行等操作

  • 从普通模式进入插入模式：

    • i：在光标的前面进行插入
    • a：在光标的后面进行插入
    • o：在光标所在行后面插入新一行

  • 更多插入命令（变种的 I/A/O ）：

    • I 在行首进行插入，约等于^a
    • A 在行尾进行输入，约等于 $a
    • O 在光标所在行的前面插入新一行，约等于ko

  • 插入模式切换到普通模式:

    • Esc 或者 ctrl + [

一般的流程是普通模式中移动光标或者执行命令，然后在需要插入代码的位置切换到插入模式进行编辑，编辑完成之后再切换回普通模式。就像是画家思考完成之后拿起画布画，然后再放下画笔继续思考，以此往复。

一旦形成肌肉记忆之后就可以不需要回忆那个命令是自己想要的操作了，这一点上跟从全拼输入法切换到双拼输入法的时候的感觉类似。

可视模式

  • 插入模式，最大的职责就是进行字符的插入，其余交给命令

  • x 删除对应的字符

  • d 删除指定范围的内容

    • dd：删除当前光标所在的行
    • diw：删除当前光标所在的单词
    • dt{char}：从当前位置删除到某个字符前为止

  • 可视模式基于普通模式

  • 在普通模式下使用 v 进入可视模式

    • 使用的 v 命令是进入以字符为单位的可视模式
      • V ：进入以行为单位的可视模式
      • ctrl + v：进入以列为单位的可视模式命令
    • 在普通模式中按 v 的时候，会把当前光标为主作为选区的起点，然后可以通过 h/j/k/l/w/b 等进行光标的移动，确定选区的终点

  • d ：剪切

    • 选择了要操作的范围之后，可用 d 命令来进行删除（实际上进行的是「剪切」操作）
    • 继续使用命令 p 在普通模式下进行粘贴

  • y：复制

    • yy 复制一行内容
    • 进入可视模式之后选定指定的内容进行复制，然后回到普通模式进行 p 粘贴操作

  • p ：粘贴

  • 文本对象 text object

    • 在不移动光标的情况下来操作一定区域内的内容

    • diw就使用到了文本对象

      • 拆解为d + iw
      • iw 即为文本对象，指的是当前单词

    • aw：当前单词以及空格

    • i(：括号内的内容

    • a(：括号以及括号内的内容

    • i( 和 a（ 中的括号可以换成任何成对的符号，例如 i[、i{、i" 等，表示删除成对的符号范围的内容

    • d 命令后面可以跟着文本对象，同理也可以使用 y 命令加文本对象复制指定区域内容

    • 类似，快速的复制任何成对符号中的内容（这在复制代码中的字符串时非常的有用）

    • 通过文本对象可以无需进入可视模式而进行一些简单的操作

命令行模式

  • 在普通模式中输入 : 会进入命令模式

  • :qa ：退出 Vim 编辑器

  • :q!：强制退出

  • :w、:wa：保持当前文件/保存所有文件

  • :s/old/new/g：将字符串 old 替换为 new

快捷键

命令模式下支持非常多的命令，但一般来说不需要进入命令模式就可以完成我们的编辑操作了，并且把需要经常使用的命令模式下的命令映射到某一个快捷键当中来进行使用。

例如，可以把上面的字符串的命令映射到某一个快捷键上

  • 在 ~/.vimrc 文件中添加如下内容：nnoremap <C-f> :s/
  • 在命令模式中重新载入vim 配置：:source $MYVIMRC
  • 然后就可以通过 ctrl + f 这个快捷键来进入命令模式，并且自动的输入 :s/，只需要在后面输入对应的替换的正则表达式就可以了

nnoremap 为普通模式下的非递归映射（此外还有 inoremap，vnoremap 等表示在不同模式下的非递归映射）：

  • 递归映射：快捷键的映射会被其他的映射所影响
  • 非递归映射：按下 c 的时候会直接映射到 a 而不会继续映射到 b
  • 对于map 以及 noremap 的详细的介绍:vim 的几种模式和按键映射

Vim 中还有 Leader 键的概念，有点像按快捷键的时候的 ctrl 或者 cmd，不同的是不需要按住它。例如：nnoremap <Leader>j 2j，实现了先按 Leader 键，然后在按 j 来映射到 2j 上的功能。

  • Leader 键在默认的情况下是 ,于是默认情况上面完成了将 ,j 映射到 2j的快捷键设置
  • 可以通过如下配置将 Leader 键修改为其他的按键：let g:mapleader = "\<Space>"
